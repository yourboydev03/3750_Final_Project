Chosen Base App

We selected the Calendar App from the textbook. It includes features such as selecting months and years from the range 1990 to 2020 and then
displaying a calendar of that specific month. We plan on increasing the calendar range to 1970 to 2050.

Project Overview

Our application will allow users to view and manage schedules, appointments, or other events. Users can create, edit, and delete events with specific dates,
times and details. This app can be utilized by a variety of people, including those individuals who want to manage their time effectively, businesses that
want to schedule meetings or manage appointments, and students who wish to control their coursework.

Proposed Enhancements

In addition to implementing the base app, we plan to make the following improvements and changes:

  • HTML:
    • Add forms/pages for actions such as event creation, event editing, and creating/editing user profiles.
    • Improve layout and structure using semantic HTML5 elements (e.g. <header>, <nav>, <main>, <section>, <footer>).
  • CSS:
    • Implement a responsive design using Flexbox/CSS Grid to ensure optimal viewing on mobile, tablet, and desktop.
    • Add a dark mode toggle that allows users to switch between light and dark themes.
    • Have different themes for the calendar to showcase customization options.
  • JavaScript:
    • Implement client-side form validation for event creation and editing to prevent incorrect inputs.
    • Use AJAX to update calendar events dynamically without a full page reload.
    • Incorporate libraries for improved date handling and more fluid user interactions.
    • Users can select themes through a list format.
  • PHP:
    • Enhance the backend logic to support robust event management, including data sanitization, error handling, and session management.
    • Calendar customization saves for users even when logging out of the app.
  • Database:
    • Expand the database schema with additional tables such as user accounts, event categories, or tags.
  • Authentication:
    • Implement a secure login and logout system using PHP sessions with password hashing to securely store credentials.
    • Add a user registration process complete with email verification and password recovery functionality.
    • Login system should only allow users to access and manage their own data
    • Logged-in users can only view and create their own events.
